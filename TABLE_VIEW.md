# 📊 表格视图说明

## 当前设计

网站现在使用**表格形式**显示所有读书俱乐部活动，不包含任何图片。

### 表格列

| 列名 | 说明 |
|------|------|
| **Event Name** | 活动名称，点击可查看详情 |
| **Type** | 在线/线下标签（蓝色 ONLINE / 绿色 IN PERSON）|
| **Library** | 图书馆名称 |
| **Date & Time** | 日期和时间（如：Tue · Jan 23 · 7:00 PM）|
| **Book** | 讨论的书籍名称和作者（如果有）|
| **Details** | "View →" 链接到详情页 |

### 功能特点

✅ **移动友好**
- 表格可以横向滚动
- 在手机上也能完美显示

✅ **搜索和筛选**
- 搜索框：搜索活动名、图书馆、书名
- 快速筛选：Today, Tomorrow, This Week, This Month, Next Month
- 高级筛选：在线/线下、按州筛选

✅ **性能**
- 无图片加载，速度非常快
- 1400+ 活动瞬间加载

✅ **表格特性**
- 鼠标悬停高亮行
- 表头固定（滚动时始终可见）
- 清晰的边框和间距

### 与原卡片设计的区别

| 特性 | 卡片设计 | 表格设计 |
|------|---------|---------|
| 视图 | 网格卡片 | 表格行 |
| 图片 | 有活动图片 | 无图片 |
| 信息密度 | 低（每卡片占据大空间）| 高（每行紧凑）|
| 适合场景 | 浏览、发现 | 快速查找、比较 |
| 移动端 | 响应式网格 | 横向滚动 |

## 为什么改用表格？

1. **图片链接问题** - RSS 提供的图片链接无法外部访问
2. **信息密度** - 表格可以在一屏显示更多活动
3. **扫描效率** - 用户可以快速扫描和比较活动信息
4. **性能** - 无需加载图片，网站打开更快

```bash
python3 -m http.server 8000

# 然后访问
http://localhost:8000
```

## 详情页

点击任何活动名称或 "View →" 链接会打开详情页，显示：
- 完整活动标题
- 图书馆信息
- 完整日期和时间
- 地点详情（或虚拟活动说明）
- 活动描述
- 书籍信息（如果有）
- 报名信息
- 分类标签
- 链接到图书馆原始页面
